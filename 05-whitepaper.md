## 执行摘要

Solana 已成为 AI Agent 自主交易的核心战场。2025 年，Solana 上 AI Agent 驱动的 DEX 交易量达 **310 亿美元**；2026 年第一季度，Solana 占全球 x402 AI 支付总量的 **82%**。AI Agent 正以前所未有的速度在链上自主操作资金。

然而，这片高速成长的生态面临一个根本性危机：**信任缺失**。

在 Solana 主网，任何人只需 5 分钟、花费 1 美元，就能部署一个 AI Agent 开始自主操作用户资金。没有审核、没有历史记录、没有行为证明。当你委托 Agent 将 10,000 USDC 兑换为 SOL 时，你无法知道：它历史上做过什么？这次任务它真的按你的要求做了吗？

**AgentProof 是这个问题的答案**。

AgentProof 是 Solana 上第一个专注于 **AI Agent 全生命周期行为信任**的链上协议，提供：

- **注册即审计**：Agent 注册时，自动拉取历史链上数据，LLM 识别真实意图，生成初始信用分和安全指数
- **接入即保障**：一行 SDK 接入，链上条件结算，资金全程由合约托管，Agent 无法越权
- **执行即证明**：每笔任务铸造 ProofNFT，链上永久存证，信用分 SBT 持续更新
- **持续风险监控**：实时检测异常行为，自动冻结并告警

AgentProof 的长期愿景是成为 **Solana Agent 生态的 Apple App Store**——上架要审核，有评分，付款有保障，让整个 Agent 经济建立在可验证的信任基础之上。

## 一、背景与问题

### 1.1 AI Agent 经济的崛起

过去两年，大型语言模型（LLM）能力的爆炸式提升催生了一个全新的经济形态：**AI Agent 经济**。AI Agent 不再只是回答问题的工具——它们开始自主管理资金、执行交易、参与 DeFi 协议、代表用户完成复杂的链上操作。

Solana 成为这场革命的核心战场，原因显而易见：

- **极低延迟**：400 毫秒区块时间，AI Agent 的实时决策得到即时链上确认
- **极低费用**：单笔交易成本 < $0.001，AI Agent 可以高频操作而不产生摩擦成本
- **成熟的 DeFi 生态**：Jupiter、Raydium、MarginFi 等协议提供丰富的金融原语
- **x402 原生支持**：HTTP 402 支付协议在 Solana 上的深度集成，让 Agent 间支付成为基础设施

**数据支撑**：
- 2025 全年，Solana AI Agent 驱动的 DEX 交易量：**$310 亿**
- x402 协议 2025 年 5 月至 2026 年 3 月：累计 **1.5 亿笔**交易，交易量超 **$5,000 万**
- 2026 年 2 月：Solana 占全球真实 x402 交易量的 **82%**
- 当前全球 AI Agent 市值中，Solana 占据 **56.48%**（约 $84 亿）

### 1.2 信任基础设施的缺失

当 Agent 经济以几何级数增长时，一个根本性的基础设施空白暴露出来：**没有人知道链上的 Agent 是否可信**。

```
Solana 主网准入门槛：

  生成一个 Ed25519 Keypair          → 1 秒
  充入 0.01 SOL（约 $1）            → 30 秒
  部署一个 Agent 开始在链上交易      → 5 分钟

链上当前现状：
  → 数千个钱包地址正在自动发送 swap / 转账 / 质押交易
  → 你无法知道：背后是人还是 AI Agent？
  → 你无法知道：这个 Agent 历史上做过什么？
  → 你无法知道：它正在做的是你委托的事情吗？
```

**真实的用户困境**：

> 用户委托 Agent：「把我账户里的 1000 USDC 换成 SOL，滑点不超过 1%」
>
> Agent 报告：「已完成」
>
> 用户无法验证：
> - 滑点是否真的没超 1%？
> - Agent 有没有把部分资金转给自己？
> - 这个 Agent 历史上有没有欺诈过其他用户？
> - 它现在的行为和它声称的功能一致吗？

### 1.3 现有基础设施的空缺

| 基础设施 | 解决的问题 | 遗留问题 |
|---------|-----------|---------|
| **Solana 公链** | 链上执行能力 | 不区分 Agent 与人类，无信任层 |
| **x402 / Latinum** | Agent 支付轨道 | 只解决「钱付了没」，不解决「事做了没」|
| **ElizaOS / Solana Agent Kit** | Agent 开发框架 | 无行为验证机制 |
| **BlockHelix** | 支付收据注册 | 金融层（钱的流向），非执行层（行为真实性）|
| **SATI** | Agent 历史能力评分 | 静态历史评分，无实时执行验证 |
| **proof-of-agent** | ZK 执行证明 | 通用 ZK，无信用体系，无 x402 集成 |

**核心空白**：没有协议同时提供「历史行为审计 + 实时执行验证 + 链上信用积累 + 条件支付释放」的完整信任闭环。

## 二、解决方案：AgentProof 协议

### 2.1 设计原则

> **不依赖 Agent 诚实，在结构上使作恶无利可图。**

AgentProof 的核心设计哲学是：信任不是依赖 Agent 主观诚实，而是通过协议结构让作恶行为在技术上受限、在经济上亏损。

**四条设计原则**：

1. **链上不可篡改**：所有验证结果、信用变化、历史记录永久存储在 Solana 公链，任何人可查
2. **结构性安全**：Escrow 托管 + Capability Manifest + 质押杠杆倍数，让作恶在结构上无法完成
3. **最小接入摩擦**：一行 SDK 代码，Agent 开发者无需改变业务逻辑
4. **经济激励对齐**：风险监控节点、Agent、用户三方的经济利益共同指向「正确行为」

### 2.2 三阶段信任体系

#### 阶段一：注册 + 历史审计（初始信任建立）

Agent 在 AgentProof 注册时，系统自动执行全量历史链上数据分析：

```
输入：
  Agent 钱包公钥（必填）
  功能声明：「我是 DeFi 交易 Agent，执行 swap / 套利」
  质押保证金（SOL）

自动执行：
  1. Helius API 拉取全量历史 TxSig（最近 500-1000 笔）
  2. 解析每笔交易：调用的合约、资金流向、参数
  3. LLM（Claude）意图识别：
     → 该 Agent 历史上主要做了什么？
     → 声称的功能和实际行为是否匹配？
     → 有没有从未声明的异常操作？
  4. 生成初始评分：
     → 信用分（Credit Score）0-100
     → 安全指数（Safety Index）0-100
     → 风险标记列表
  5. 铸造初始 SBT（灵魂绑定 Token）

新 Agent（无历史记录）：
  → 信用分从基准分（50 分）开始
  → 质押越高，基准分越高
  → 每完成一笔经过 AgentProof 验证的任务，信用分上升
```

**关键差异**：SATI 等竞品做的是「能力评分」（Agent 能做什么），AgentProof 做的是「行为审计」（Agent 实际上做了什么）——两者互补，但后者是信任基础中更根本的一层。

#### 阶段二：接入保障（执行过程中的信任保障）

Agent 集成 AgentProof SDK 后，每次执行任务走以下流程：

```
用户设定意图参数 + 资金锁定到 Escrow PDA
         ↓
Agent 开始任务前提交「盲签名承诺」（防止事后选择性提交）
         ↓
Agent 执行链上操作（swap / 质押 / 转账）→ TxSig 产生
         ↓
AgentProof.execute({ taskId, jupiterRouteParams })  ← 一行代码
         ↓
合约 CPI → Jupiter 执行 swap（原子交易，400ms）
         ↓
合约读取实际执行结果，数学条件核对（无需任何第三方）
         ↓
合约核对：实际执行参数 vs 用户意图条件
  ├── ✅ 达成 → ProofNFT 铸造 + 信用分上升 + x402 释放报酬
  └── ❌ 未达成 → Slash 质押金 + 退款用户 + 信用分下降 + 风险标记
```

#### 阶段三：持续风险监控

AgentProof 在 Agent 接入后提供实时链上行为监控：

- **滑点异常检测**：Agent 声称滑点 ≤ 1%，实际超标立即告警
- **资金流向监控**：执行 swap 之外有无额外资金转出？有无向黑名单地址转账？
- **合约交互黑名单**：是否与已知钓鱼 / 恶意合约交互？
- **行为模式异常**：突然大量小额转账（洗钱模式）？深夜高频操作（被盗私钥特征）？
- **自动保护**：高风险行为 → 自动冻结托管报酬 + 触发仲裁 + 通知用户

### 2.3 四层资金安全体系

> 核心安全问题：**如果 Agent 信用分很高但质押只有 0.5 SOL，它能在执行任务时把用户价值 $10,000 的资金转走吗？**

**第一层：Solana 基本规则**
```
用户账户的资金 = 需要用户私钥签名才能动
Agent 没有用户私钥 → 直接盗取用户账户在技术上不可能
```

**第二层：Escrow 资金托管**
```
错误的资金流：用户 → Agent 直接操作 → 可能转走

AgentProof 的安全资金流：
  用户 → $10,000 USDC → TaskEscrow PDA（合约控制）
  Agent 在外部执行 swap（接受合约指令）
  验证通过 → 合约将 SOL 打到用户地址，报酬打给 Agent

关键：Agent 全程无法触及 Escrow 里的 $10,000 USDC
```

**第三层：Capability Manifest 合约强制执行**
```
Agent 注册时声明能力边界（合约强制，不是自律）：
  allowed_programs: [JupiterV6, Raydium]
  max_single_tx_usd: 500
  allowed_output_tokens: [SOL, USDC, USDT]

超出声明范围的操作 → 合约层直接拒绝
即使 Agent 有用户授权，也只能在 Manifest 内操作
```

**第四层：质押杠杆倍数限制**
```
任务最大价值上限 = Agent 质押金额 × 杠杆倍数

杠杆倍数由信用分决定：
  信用分 0-50   → 5x  （新 Agent，质押 1 SOL → 最大接 $750 任务）
  信用分 51-75  → 20x （质押 1 SOL → 最大接 $3,000 任务）
  信用分 76-90  → 50x （质押 1 SOL → 最大接 $7,500 任务）
  信用分 91-100 → 100x（质押 1 SOL → 最大接 $15,000 任务）

示例：Agent 质押 0.5 SOL（约 $75），信用分 80
  → 最大任务价值：$75 × 50 = $3,750
  → 用户想委托 $10,000 任务 → 合约直接拒绝
```

**兜底层：Slash + 信用分归零**

前四层防御使作恶在技术和结构上极难发生。即使突破前四层，Slash 机制作为最后兜底：扣除全部质押 + 信用分归零 + 永久风险标记。对长期运营的 Agent 而言，未来的信誉价值远超任何一次作恶收益。

### 3.2 意图验证技术方案

AgentProof 专注于「可量化意图」的链上验证，排除无法客观核查的「语义意图」：

| 类型 | 示例 | AgentProof 处理方式 |
|------|------|-------------------|
| ✅ 可量化意图 | swap：输入/输出代币/滑点/金额 | 链上数学比对 |
| ✅ 可量化意图 | 转账：目标地址/金额/代币 | 链上账户字段核查 |
| ✅ 可量化意图 | 质押：协议/金额/期限 | 链上状态验证 |
| ❌ 语义意图 | 「帮我找最好的投资机会」 | 不做（无法客观核查）|
| ❌ 语义意图 | 「根据市场情绪决策」 | 不做（排除在外）|

**执行验证流程**（无 LLM，纯数学，由 Solana 运行时保证）：
```
合约 CPI 原子执行：
  1. 快照执行前账户余额
  2. CPI → Jupiter V6 执行 swap
  3. 读取执行后余额，计算实际滑点
  4. 核对 MaxSlippage：实际滑点 ≤ 用户设定上限？
  5. 核对 MinOutput：输出金额 ≥ 最低要求？
  6. 核对 Deadline：在截止时间内？
  → 全部满足：结算 + ProofNFT；任一不满足：revert + Slash
```

### 3.3 信用评分算法（EWMA）

借鉴 SAEP 协议的 6 维 EWMA 设计，AgentProof 采用以下评分模型：

**初始信用分（注册时 LLM 历史审计计算）：**
```
initial_credit = (
  historical_completion_rate × 0.35 +   // 历史任务完成率
  fund_safety_record × 0.35 +            // 资金安全记录
  intent_consistency × 0.30             // 声明与行为一致性
) × 100
```

**动态更新（每次任务完成）：**
```
new_score = 0.80 × old_score + 0.20 × task_result_score

task_result_score：
  完美完成（条件全满足）→ 100
  部分完成            → 按比例
  失败（被 Slash）     → 0
  超时                → 30
```

**α = 0.20 的设计意义**：历史权重 80%，新样本权重 20%。这意味着：
- 单次成功不能大幅提升信用分（防止短期刷分）
- 历史积累的声誉高度稳定（真正的长期信用资产）
- 一次作恶不会让信用分立刻崩溃，但持续作恶会指数级下降

## 七、风险分析

### 7.1 技术风险

| 风险 | 描述 | 缓解方案 |
|------|------|---------|
| **见证节点串谋** | ~~已移除~~，CPI 原子执行无需信任任何第三方 | — |
| **LLM 意图误判** | Claude 对历史交易意图分类不准确 | 人工复审高分 Agent；持续迭代 Prompt |
| **合约漏洞** | Anchor 合约存在可利用的 Bug | 专业 Solana 安全审计（上线前）；漏洞赏金计划 |
| **链上拥堵** | Solana 网络拥堵影响 CPI 执行时效 | 使用优先费机制；CPI 原子执行本身无额外网络依赖 |

### 7.2 市场风险

| 风险 | 描述 | 缓解方案 |
|------|------|---------|
| **冷启动** | Agent 开发者初期不愿意注册 | 前 50 个 Agent 免费注册；共建声誉激励 |
| **大平台自建** | Jupiter / Raydium 自己做 Agent 信任层 | 协议中立性是优势；先发优势 + 数据壁垒 |
| **竞品追赶** | BlockHelix 等扩展到行为验证 | 持续迭代；历史审计数据积累是护城河 |
| **监管不确定性** | DeFi / AI 监管趋严 | 协议中立（只做验证，不做资产管理）；合规友好设计 |

### 7.3 经济风险

| 风险 | 描述 | 缓解方案 |
|------|------|---------|
| **质押价值波动** | SOL 价格下跌削弱质押保障价值 | 引入稳定币（USDC）作为质押选项 |
| **手续费过低** | 0.1% 手续费在任务量不足时收入微薄 | 订阅制 + 企业 API 覆盖固定成本 |
| **Token 价值归零** | $PROOF 发行后无人持有 | Token 发行在协议有真实收入后（不依赖 Ponzi）|

## 十、结语：为什么是现在？

AI Agent 经济的临界点已经到来。

2025 年，链上 Agent 交易量 $310 亿；2026 年，这个数字还在以月均 30% 的速度增长。但信任基础设施的缺失正在成为整个生态规模化的瓶颈——用户不敢把大额资金交给 Agent，优质 Agent 无法证明自己的价值，恶意 Agent 和优质 Agent 在链上看起来一模一样。

AgentProof 是填补这个空白的解决方案。

我们不是做另一个支付协议，不是做另一个 Agent 框架。我们做的是整个 Solana Agent 经济赖以运转的**信任基础设施**——注册即审计，接入即保障，执行即证明。

Solana 上的每个 Agent，都需要 AgentProof。
