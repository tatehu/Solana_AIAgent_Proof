# AgentProof Score Platform — 产品设计文档

**日期**：2026-04-25
**项目**：AgentProof Score Platform
**状态**：设计确认，待实现

## 一、产品定位

AgentProof Score Platform 是一个面向 Solana AI Agent 生态的信誉评分与保险平台。

**核心策略**：评分体系免费开放引流，保险产品作为转化变现层，SDK 接入作为 B 端长期收入。三者形成自增强飞轮。

**目标用户**：
| 角色 | 诉求 | 平台价值 |
|------|------|---------|
| Agent 使用者 | 找到安全可用的 Agent | 排行榜发现 + 保险兜底 |
| Agent 开发者 | 让更多用户使用自己的 Agent | 高评分 = 低保费 = 更多用户 |
| 流动性提供者 | 赚取保费收益（未来） | 互助池阶段注入流动性 |

## 二、整体架构

```
┌─────────────────────────────────────────────────────┐
│  变现层   保险购买 / SDK 接入费 / 数据订阅              │
├─────────────────────────────────────────────────────┤
│  产品层   Agent 排行榜 / 评分详情页 / 保险购买流程       │
├─────────────────────────────────────────────────────┤
│  评分层   行为分析引擎 / 信誉评分模型 / 保费定价模型      │
├─────────────────────────────────────────────────────┤
│  数据层   链上爬虫 / ElizaOS & Agent Kit 索引 / Helius │
└─────────────────────────────────────────────────────┘
```

**增长飞轮**：
```
用户来逛排行榜
      ↓
发现心仪 Agent → 点跳转链接使用
      ↓
顺便购买保险（保费基于评分定价）
      ↓
出险数据 → 反哺评分模型（更准）
      ↓
Agent 开发者为降低用户保费 → 主动接入 SDK 提升评分
      ↓
更多 Agent 接入 → 平台评分更权威 → 更多用户来逛
```

## 三、Agent 数据采集范围

**采集策略：精准圈定（B 策略）**

覆盖已知 Agent 框架注册的 Agent，优先索引：
- ElizaOS 框架部署的 Agent
- Solana Agent Kit 构建的 Agent
- GOAT Framework Agent
- 其他可识别框架（通过链上部署模式识别）

**数据来源**：
- Helius API（交易历史、账户数据）
- 链上 Capability Manifest（AgentProof SDK 已接入的 Agent）
- 框架官方注册表（ElizaOS Agent Registry 等）

**冷启动：** 新 Agent 无历史数据时默认 50 分（B 级），通过行为积累提升。

## 四、评分模型

### 4.1 评分维度（100 分制）

| 维度 | 权重 | 评分依据 |
|------|------|---------|
| 历史行为安全性 | 35% | 异常操作占比、黑名单地址交互、滑点超标频率 |
| 任务完成率 | 25% | 声明任务 vs 实际执行成功率（链上可验证） |
| 资金风险暴露 | 20% | 单笔最大操作金额、与未知合约交互频率 |
| 框架合规性 | 12% | 是否接入 AgentProof SDK、Capability Manifest 完整性 |
| 活跃度衰减 | 8% | 长期不活跃后重新活跃的信用衰减（防养号套现） |

> **注**：维度名称公开，具体权重算法在链下服务保密，防止 Agent 针对性绕过检测。

### 4.2 评级与保费系数

| 分数区间 | 评级 | 保费系数 |
|---------|------|---------|
| 90–100 | ★★★★★ AAA | × 0.5 |
| 75–89 | ★★★★☆ AA | × 0.8 |
| 60–74 | ★★★☆☆ A | × 1.0（标准） |
| 45–59 | ★★☆☆☆ B | × 1.5 |
| < 45 | ★☆☆☆☆ C | 不可投保 |

### 4.3 新 Agent 快速升分机制

| 动作 | 加分 |
|------|------|
| 接入 AgentProof SDK | +10 分 |
| 完整填写 Capability Manifest | +5 分 |
| 前 30 天无异常交易（每周） | +2 分（最多 +8 分）|

## 五、前端产品设计

### 5.1 排行榜页（首页 / 引流核心）

- 表格展示：排名、Agent 名称、评级星级、评分、任务完成率
- 筛选项：框架类型、评级、Agent 类型、钱包地址搜索
- 每行操作按钮：**[使用]**（跳转至 Agent 官方入口，含 ref 追踪）、**[保险]**（原地弹窗投保）

### 5.2 Agent 详情页（信任建立）

- 头部：Agent 名称、评级、总分、框架、钱包地址、活跃天数
- 评分详情：各维度得分 + 90 天评分走势折线图
- 交易摘要：近 100 笔交易中异常数、平均滑点、最大单笔金额
- 链上数据链接：跳转 Solana Explorer 验证原始数据
- 行动按钮：[立即使用 →] [购买保险 🛡️]

### 5.3 保险购买流程（目标 30 秒完成）

```
第1步：选择保障方案
  资金安全险 / 执行保障险 / 双险套餐（推荐）

第2步：确认绑定信息
  绑定目标 Agent 地址 + 用户钱包地址

第3步：链上签名 → 保单 NFT 发到用户钱包 → 保单生效
```

## 六、保险产品设计

### 6.1 双险产品规格

| | 资金安全险 | 执行保障险 |
|---|---|---|
| **保障范围** | Agent 操作导致资金损失 | Agent 未完成约定任务 |
| **触发条件** | 单笔损失 > 保额 10%，链上可验证异常 | 任务超时未执行 / 执行失败有链上记录 |
| **赔付比例** | 损失金额的 80% | 手续费 / 服务费 100% 退还 |
| **MVP 最高保额** | 5 SOL | 0.5 SOL |
| **定价公式** | 保额 × 1% × 风险系数 | 固定 0.02 SOL / 次 |
| **双险套餐** | 两险合并，保费 × 0.8 折扣 | 同左 |

### 6.2 理赔流程（链上自动核验）

```
用户提交理赔申请
      ↓
系统自动拉取链上交易记录核验（Helius API）
      ↓
与保单 NFT 中记录的条款比对
      ↓
符合条件 → 自动从风险储备金赔付（24 小时内）
不符合条件 → 返回拒赔理由（链上可查）
```

### 6.3 风险储备金管理（MVP 阶段）

| 项目 | 规则 |
|------|------|
| 初始注入 | 团队/投资方注入 50 SOL 启动 |
| 保费分配 | 80% 进储备金，20% 为平台利润 |
| 单日赔付上限 | ≤ 储备金总量 10%（防挤兑） |
| 透明公示 | 储备金余额实时链上可查 |
| 未来升级 | 有规模后去中心化为链上互助池（Nexus Mutual 模式） |

## 九、核心风险与应对

| 风险 | 描述 | 应对 |
|------|------|------|
| 储备金耗尽 | 早期出险集中，储备金不足 | 单日赔付上限 10% + 保额上限 5 SOL 控制风险敞口 |
| 评分被操纵 | Agent 刻意刷高评分 | 权重保密 + 活跃度衰减机制 + 行为组合检测 |
| Agent 数据不足 | 新 Agent 无历史数据 | 快速升分机制 + 接入 SDK 激励 |
| 理赔纠纷 | 用户认为符合条件但被拒赔 | 链上数据全程可查，拒赔理由公开透明 |
| 冷启动流量 | 无 Agent 数据吸引不了用户 | 先索引存量知名 Agent 建立内容，再做增量 |
